# 任务文档协议

本文件定义 `/opencat-work` 对项目内真实 `TODO.md` 与 `DONE.md` 的读取、判断和回写规则。`${CLAUDE_SKILL_DIR}/template/TODO.md` 与 `${CLAUDE_SKILL_DIR}/template/DONE.md` 仅用于格式参考，属于只读模板。

## 权威来源

- 项目内真实 `TODO.md` 决定当前有哪些任务可执行
- 项目内真实 `DONE.md` 记录已完成任务的精简流水
- Git 仓库当前状态是第三个权威信号，用于判断是否仍有未收尾工作

## 项目内 `TODO.md` 解析规则

### 优先级

- 优先级固定为 `P1 > P2 > P3`

### 活跃章节

- 章节标题带 `>`，例如 `## P1 >`
- 含义：该章节被显式授权进入执行队列

### 活跃任务

- 任务行带 `>`，例如 `- > 修复登录异常`
- 含义：该任务是当前执行项

### backlog

- 章节无 `>` 且任务行也无 `>`
- 含义：仅是待办记录，不得被自动执行

## 状态所有权

- 章节标题上的 `>` 由用户或上游流程显式维护
- `/opencat-work` 不得自动新增、删除或改写章节标题上的 `>`
- 任务行上的 `>` 可在已激活章节内被本技能维护

## 项目内 `TODO.md` 允许的写回范围

只允许以下三类修改：

1. 删除已完成任务行
2. 在已激活章节内维护任务行上的 `>`
3. 在失败任务行后追加注释，例如 `<!-- 失败: 原因 -->`

除此之外，尤其是章节标题文本，必须逐字保持不变。

## 保存前校验

每次写回项目内真实 `TODO.md` 前，必须：

1. 提取原始章节标题列表
2. 提取修改后的章节标题列表
3. 逐行比较
4. 若任意标题发生变化，立即放弃本次写回，并改用更小粒度编辑

## 项目内 `DONE.md` 记录格式

推荐格式：

```text
- [时间] 任务名称 — 变更
```

若需更严格控制，也可使用：

```text
- [时间（分钟）] <任务名称> — <change-name>
```

要求：

- 保持精简
- 不写长段说明
- 一行一条记录

## 修复类任务

任务名以“修复”开头时，必须重新验证当前现状，不得因项目内 `DONE.md`、archive 或历史相似记录就直接跳过。
