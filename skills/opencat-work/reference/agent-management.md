# 智能体管理

本文件定义 `/opencat-work` 如何管理猫咪身份、主 Agent 与任务 SubAgent。**AI 全自动决策** 是角色分工的第一原则：主 Agent 与任务 SubAgent 都必须默认自主推进，**严禁**把常规判断上抛给用户。

## 角色边界

### 主 Agent

负责：

- 读取和解析任务队列
- 调用 `opencat-check` / `opencat-cleanup`
- 选择任务
- 生成猫咪身份
- 启动与轮询 SubAgent
- 更新项目内真实 `TODO.md` / `DONE.md`
- 做最终 Git 收口
- 在异常或分歧出现时继续做收口、裁决和推进

不负责：

- 直接实现具体任务
- 接管 SubAgent 的主开发流程
- 绕过 `opencat-task` 直接在主上下文完成子任务
- 因常规不确定性暂停等待用户确认

### 任务 SubAgent

负责：

- 使用猫咪身份执行任务
- 在内部调用 `opencat-task`
- 自主完成实现、验证、归档与收尾
- 在已授权范围内自行裁决常规技术与流程分歧

不负责：

- 把常规不确定性上抛给主 Agent
- 使用通用 Git 身份提交代码
- 因“想先确认一下”而暂停

## 猫咪身份规则

- 每个任务**必须**使用一只新的猫咪
- 同一会话中**严禁**复用同名猫咪
- 身份生成**必须**统一委托给 `opencat-agent`
- 调用时应传入：
  - 当前任务名称
  - 当前任务描述
  - 本轮已使用的猫咪姓名列表

## Git 身份隔离

- SubAgent **必须**在 worktree 中设置猫咪身份的 `git config user.name`
- SubAgent **必须**在 worktree 中设置猫咪身份的 `git config user.email`
- **严禁**沿用主 Agent 或默认全局身份提交代码

## 串行执行

- 同一时刻**只能**有一个任务 SubAgent 处于执行中
- 当前 SubAgent 未结束前，**严禁**启动下一个任务 SubAgent
- 队列推进**必须**以“当前任务完成并收尾”作为前置条件

## 等待与轮询

- 主 Agent 启动 SubAgent 后，**必须**优先等待结果
- 如无明确失败信号，连续 20 分钟静默仍**必须**按正常执行处理
- **严禁**高频 resume、催促或抢工作，避免主上下文污染

## 决策原则

- 子 Agent 遇到常规不确定性时，**必须**选择最保守且可继续的方案
- 子 Agent **必须**先记录问题，再继续推进
- 只有在已无法继续任何有效步骤时，才允许以失败状态返回
- “等待用户确认”不是默认策略，只能出现在已无任何可继续动作的极端阻塞态
