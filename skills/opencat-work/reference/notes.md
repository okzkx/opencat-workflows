# 补充注意事项

本文件用于存放不适合进入“最高准则”或“主工作流”但仍需长期保留的补充说明。

## 关于最高工作守则

- AI 全自动决策并持续推进是本技能的最高工作守则
- 该守则只在已显式授权的任务范围内生效，不允许借此擅自扩大执行范围
- 常规不确定性、短时静默、局部异常、主观犹豫都不是暂停理由

## 断点恢复

- `/opencat-work` 支持从中断位置继续
- `/clear` 后恢复时，**必须**优先依据项目内真实 `TODO.md`、`DONE.md` 和 Git 状态重建上下文

## 关于未提交改动

- 当前未提交改动默认视为允许自动收口的工作流残留
- 不因这类改动单独中断整个队列
- 但若判定为不明来源异常，仍要先独立提交收口

## 关于 backlog

- 未激活 backlog 不是当前执行范围
- `/opencat-work` 不负责替用户决定哪些 backlog 章节应激活
- 自动决策能力只用于“已激活任务如何推进”，不用来“替用户扩大授权范围”

## 关于“修复”类任务

- “修复”类任务天然带有验证义务
- 需要重新确认 bug 是否仍存在、现有实现是否已覆盖、是否有回归路径

## 关于长时间静默

- 长时间静默不等于卡死
- 如无明确失败信号，必须按“仍在执行”处理

## 关于文档维护

- 若后续继续扩展规则，**必须**优先更新对应 `reference/` 模块
- 仅当规则影响主入口判断时，才同步回主 `SKILL.md`
- `${CLAUDE_SKILL_DIR}/template/TODO.md` 与 `${CLAUDE_SKILL_DIR}/template/DONE.md` 仅作模板，不应被当作运行时任务文件
