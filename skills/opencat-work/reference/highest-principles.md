# 最高准则

本文件用于展开 `/opencat-work` 的最高优先级规则，帮助维护者统一裁决冲突场景。运行时如遇细节冲突，以这里的规则为最高判断依据。**AI 全自动决策并持续推进** 是本技能的第一原则，优先级高于一般流程便利性或交互偏好。

## 核心目标

- **必须**在已授权任务范围内执行 AI 全自动决策、全自动推进
- **必须**完成整个 OpenCat Work 队列的可执行部分
- **必须**把当前仓库状态视为现实输入，而不是暂停理由
- **必须**优先维持 worktree 状态机、任务隔离和最终收口的完整性

## 不可违反规则

1. **最高守则**：AI 在已授权任务范围内**必须**自主决策并持续推进；除非已无法继续任何有效动作，否则**严禁**暂停等待用户确认。
2. **必须**先 `opencat-check`，再 `opencat-cleanup`，通过闸门后才能领取新任务。
3. **必须**只消费显式激活的任务；不得把普通 backlog 自动转成执行项。自动决策只允许发生在已授权范围内。
4. **必须**通过 SubAgent 调用 `opencat-task` 执行任务；主 Agent **严禁**因为主观判断“更快”而接管实现。
5. **必须**保证同一时刻只有一个任务 SubAgent 在运行。
6. **必须**把章节标题上的 `>` 当作只读授权信号；不得私自改写。
7. **必须**在异常变更出现时先独立收口，再继续原流程；**严禁**把异常当作默认暂停理由。
8. **必须**在全流程结束后统一执行最终 cleanup 与 Git 收口。

## 冲突裁决顺序

当不同说明出现冲突时，使用以下顺序裁决：

1. 最高准则
2. 主 `SKILL.md` 中的工作流摘要
3. `reference/` 下的详细说明
4. 模板文件中的示例格式

若某条说明会导致“暂停等待确认”，而另一条说明允许“继续推进并记录问题”，则**必须**选择后者。

## 何时允许停止

只有满足以下条件时才允许停止本技能：

- 关键命令真实失败
- 已尝试可行的后续收口或绕行步骤
- 当前仓库状态已经无法继续推进任何有效动作
- 当前已不存在任何可继续的记录、归档、清理、提交、汇报或下一步推进动作

若仍能继续记录、归档、清理、提交或推进下一步，就不应停止。
