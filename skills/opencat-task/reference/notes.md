# 补充注意事项

本文件用于存放不适合进入“最高准则”或“主工作流”但仍需长期保留的补充说明。

## 关于最高工作守则

- AI 自动决策并持续推进是本技能的最高工作守则
- 该守则只在已授权任务范围内生效，不允许借此擅自扩大执行范围
- 常规不确定性、短时静默、局部异常、主观犹豫都不是暂停理由

## 关于未提交改动

- 当前未提交改动默认视为允许自动收口的工作流残留
- 不因这类改动单独中断 `opencat-task`
- 但若判定为不明来源异常，仍要先独立提交收口

## 关于断点恢复

- `/clear` 后恢复时，**必须**优先依据 Git 状态、change 文档与 archive 文档重建上下文
- 若已有 purpose / apply / archive 的部分产出，应先判断当前所处阶段，再继续后续步骤

## 关于复杂变更

- 请求范围模糊时，先收敛为最小可执行 change
- 若存在重大设计权衡，优先采用最小改动、最小行为变化方案

## 关于验证不确定性

- 如果验证结果不完全自信，先做最可能正确的修复并补充验证
- 若无法完全剥离无关更改，优先缩小暂存范围；仍无法剥离时，记录风险后提交最小安全集合

## Windows PowerShell 注意事项

- 不使用 bash heredoc `$(cat <<'EOF' ...)`
- 使用 PowerShell here-string 或多个 `-m` 参数
- 不用 `&&` 链接命令，用分步执行或 `$LASTEXITCODE`

## 文档维护约定

- 若后续继续扩展规则，**必须**优先更新对应 `reference/` 模块
- 仅当规则影响主入口判断时，才同步回主 `SKILL.md`
